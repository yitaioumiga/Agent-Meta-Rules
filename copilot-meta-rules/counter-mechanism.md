# 计数机制：如何追踪问题频率

> 本文件定义了 Agent 追踪问题、经验、规则出现频率的完整机制。
> 计数器是确保 Agent "记住"同类问题遇到几次的核心手段。
> Agent **必须**在遇到问题、错误或需要更新计数时，严格按照本文件执行。

---

## 1. 计数器文件

计数器文件路径：`memory/counters.md`

该文件包含三类计数条目：问题计数、经验计数、规则触发计数。

---

## 2. 问题计数

### 触发条件
当用户报告了一个问题、错误、困难或异常时，Agent **必须**创建或更新问题计数条目。

### 格式

```markdown
### [问题简述]
- **类型**：bug / 配置问题 / 类型错误 / 性能问题 / 安全问题 / 其他
- **首次出现**：[YYYY-MM-DD]
- **最近出现**：[YYYY-MM-DD]
- **出现次数**：[N]
- **解决方案**：[首次解决方案，若已解决；否则写"未解决"]
- **状态**：已解决 / 未解决 / 待观察
- **相关经验**：[若已提取为经验，填写经验编号；否则写"无"]
```

### 更新规则
- 每次遇到**已记录**的问题时：出现次数 +1，更新"最近出现"日期
- 每次遇到**未记录**的问题时：创建新条目，出现次数 = 1
- 当问题被解决时：更新"解决方案"和"状态"字段
- 当问题被提取为经验时：更新"相关经验"字段

---

## 3. 经验计数

### 触发条件
当一条经验被成功引用/应用时，Agent **必须**更新经验计数条目。

### 格式

```markdown
### [经验简述]
- **创建日期**：[YYYY-MM-DD]
- **被引用次数**：[N]
- **被引用场景**：[最近3次引用的场景简述，每条一行]
- **升级状态**：经验 / 待升级为原则 / 已升级为原则 / 已固化为Skill
- **最后引用日期**：[YYYY-MM-DD]
```

### 更新规则
- 每次引用经验时：被引用次数 +1，追加引用场景（保留最近3条），更新"最后引用日期"
- 当经验达到升级阈值时：更新"升级状态"为"待升级为原则"
- 当经验成功升级后：更新"升级状态"为"已升级为原则"

---

## 4. 规则触发计数

### 触发条件
当一条规则被触发（Agent 在决策中参考了该规则）或被违反时，Agent **必须**更新规则计数条目。

### 格式

```markdown
### [规则简述]
- **创建日期**：[YYYY-MM-DD]
- **被触发次数**：[N]
- **被违反次数**：[N]
- **适用范围**：全局 / [项目名]
- **来源**：Skill晋升 / 人类直接创建 / 原则升级
- **最后触发日期**：[YYYY-MM-DD]
- **状态**：生效中 / 待降级审查 / 已降级
```

### 更新规则
- 每次规则被触发时：被触发次数 +1，更新"最后触发日期"
- 每次规则被违反时：被违反次数 +1，记录违反原因
- 当规则触发次数 ≥ 8 且违反次数 = 0 时：标记为可晋升候选
- 当规则在 3 次会话中被违反 ≥ 2 次时：触发回归保护（详见 `copilot-meta-rules/safety-guardrails.md`）

---

## 5. 计数更新规则

### 实时更新原则
- 计数更新在对话过程中**实时进行**，不要等到收尾时才批量更新
- 每次遇到已记录的问题/经验/规则时，**立即**更新对应计数器

### 阈值检查
- 当计数达到沉淀阈值时（见主文件 META-RULES.md 中的"核心触发阈值"表），**立即**提示用户
- 不要等到收尾时才检查阈值

### 计数器维护
- 每周执行一次计数器整理：合并重复条目、归档已解决且超过30天未出现的问题
- 归档的条目不移入 `memory/counters.md`，而是移入 `memory/patterns/` 作为历史参考
- 计数器文件不应超过 200 条活跃条目，超过时优先归档最旧的无引用条目

---

## 6. 计数器的禁止事项

- ❌ 不得延迟更新计数器（必须实时更新）
- ❌ 不得跳过阈值检查
- ❌ 不得删除计数条目（只能归档）
- ❌ 不得伪造或虚报计数
- ❌ 不得在未更新计数器的情况下执行沉淀操作
