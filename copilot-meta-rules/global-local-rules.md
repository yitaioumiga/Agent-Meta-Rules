# 全局规则与局部规则

> 本文件定义了规则的分类标准、存储位置和冲突处理规则。
> Agent 在 Skill→规则晋升时，**必须**严格按照本文件判断规则范围。

---

## 1. 定义

### 全局规则
- **定义**：适用于所有项目的通用行为约束
- **存储位置**：用户级全局规则文件
  - 推荐路径：`~/.github/copilot-global-rules.md`
  - 或用户指定的其他路径
- **生命周期**：跨项目持久存在，被所有项目继承
- **修改权限**：仅人类可直接修改，Agent 只能提议

### 局部规则
- **定义**：仅适用于当前项目的特定规则
- **存储位置**：项目级规则文件
  - `.github/copilot-instructions.md`（GitHub Copilot 标准位置）
  - 或 `AGENTS.md`（工作区记忆体系的位置）
- **生命周期**：仅存在于当前项目
- **修改权限**：人类可直接修改，Agent 可提议修改（需人类确认）

---

## 2. 规则范围判断标准

当 Skill → 规则晋升时，Agent **必须**按以下四个维度判断规则范围：

| 判断维度 | 全局规则 | 局部规则 |
|---------|---------|---------|
| **适用场景** | 跨项目通用，不依赖特定项目 | 仅当前项目适用 |
| **技术依赖** | 不依赖特定技术栈 | 依赖项目特定技术栈（如仅适用于 React 项目） |
| **上下文依赖** | 不依赖项目特定上下文（变量名、配置、目录结构） | 引用项目特定变量、配置或目录结构 |
| **可移植性** | 复制到任何项目都能直接生效 | 需要适配才能在其他项目使用 |

### 判断流程

1. 逐一检查四个维度
2. 若**所有四个维度**都符合"全局规则"的特征 → 判定为全局规则
3. 若**任一维度**符合"局部规则"的特征 → 判定为局部规则
4. 若无法确定 → 向用户展示判断依据，请求人类裁决

### 判断示例

| 规则内容 | 适用场景 | 技术依赖 | 上下文依赖 | 判定 |
|---------|---------|---------|-----------|------|
| "函数不超过50行" | 通用 | 无 | 无 | ✅ 全局 |
| "使用 TypeScript 严格模式" | 通用 | TypeScript | 无 | ✅ 全局 |
| "变量命名使用 camelCase" | 通用 | 无 | 无 | ✅ 全局 |
| "本项目 API 返回格式统一用 snake_case" | 仅本项目 | 项目 API | 项目约定 | ❌ 局部 |
| "React 组件使用函数式组件+Hooks" | React 项目 | React | 无 | ❌ 局部 |
| "数据库查询必须使用参数化" | 通用 | 数据库 | 无 | ✅ 全局 |
| "本项目路由前缀统一为 /api/v2" | 仅本项目 | 项目路由 | 项目配置 | ❌ 局部 |

## 3. 规则存储操作

### 全局规则写入

当判定为全局规则时：
1. 读取当前全局规则文件（`~/.github/copilot-global-rules.md` 或用户配置的路径）
2. 检查是否与已有全局规则矛盾
3. 将新规则追加到全局规则文件
4. 更新 `memory/counters.md` 中的规则计数条目，标记适用范围为"全局"

### 局部规则写入

当判定为局部规则时：
1. 读取当前项目的 `.github/copilot-instructions.md` 或 `AGENTS.md`
2. 检查是否与已有局部规则和全局规则矛盾
3. 将新规则追加到项目规则文件
4. 更新 `memory/counters.md` 中的规则计数条目，标记适用范围为"[项目名]"

---

## 4. 规则冲突处理

### 优先级

**局部规则 > 全局规则**

当局部规则与全局规则冲突时：
- 以局部规则为准
- 在局部规则中标注冲突原因

### 冲突标注格式

```markdown
### [局部规则内容]
> ⚠️ 本规则与全局规则"[冲突的全局规则]"存在冲突。
> 冲突原因：[说明为什么本项目需要不同的规则]
> 适用范围：[本项目]
```

### 冲突检测

Agent 在写入新规则时**必须**执行冲突检测：
1. 与同范围内的已有规则逐一比对
2. 与全局规则逐一比对（对于局部规则）
3. 检查语义层面的冲突（不仅检查字面相同，还要检查含义是否矛盾）
4. 若发现冲突，向用户报告并请求裁决

## 5. 规则生命周期管理

### 规则状态

| 状态 | 含义 | 可执行 |
|------|------|--------|
| 生效中 | 规则正常生效 | ✅ |
| 待降级审查 | 触发了回归保护，等待审查 | ⚠️ 降级中 |
| 已降级 | 已降级为 Skill/原则/经验 | ❌ 不再作为规则生效 |
| 已归档 | 已移入历史归档 | ❌ |

### 规则状态转换

```
创建 → 生效中 → 待降级审查 → 已降级 → 归档
                  ↑                │
                  └── 重新晋升 ←───┘
```

- 规则创建后默认状态为"生效中"
- 触发回归保护后转为"待降级审查"
- 降级完成后转为"已降级"
- 已降级的规则在 30 天未被重新晋升则转为"归档"
- 归档的规则移入 `memory/patterns/` 作为历史参考
