# 沉淀之路：从经验到规则的四级跃迁

> 本文件定义了知识从原始记忆逐步升级为强制规则的完整自进化路径。
> Agent 在涉及任何沉淀操作时，**必须**严格按照本文件中的规则执行。

---

## 1. 沉淀路径总览

```
原始记忆 → 经验 → 原则 → Skill → 规则
  (L0)     (L1)   (L2)   (L3)   (L4)
```

每一级跃迁都涉及三个核心动作：
- **抽象**（从具体到一般）：将具体的、情境依赖的经验提炼为通用的、情境无关的原则
- **压缩**（从冗余到精炼）：去除重复的、次要的信息，保留核心的、关键的知识
- **固化**（从建议到强制）：将可选的行为建议转化为默认的行为模式

---

## 2. 自动沉淀触发条件

### 2.1 经验提取（L0 → L1）

触发条件（满足任一即可）：
- 同一问题在计数器（`memory/counters.md`）中出现次数 ≥ 3
- 一个解决方案被成功应用且用户表示满意
- 一个错误被识别并修正，且该错误具有通用性

操作步骤：
1. 从对话中提取经验，按以下格式编写：

```markdown
### 经验 #[编号]
- **场景**：[触发场景描述]
- **问题**：[问题描述]
- **解决方案**：[方案描述]
- **适用条件**：[适用范围]
- **来源**：[对话日期]
```

2. 在 `memory/counters.md` 中创建经验计数条目：

```markdown
### [经验简述]
- **创建日期**：[日期]
- **被引用次数**：0
- **被引用场景**：[暂无]
- **升级状态**：经验
```

3. **向用户展示经验条目，请求确认**（展示格式："确认/修改/拒绝"）
4. 用户确认后，追加写入 `memory/experiences.md`
5. 用户拒绝后，记录拒绝原因（若有），不写入

### 2.2 经验 → 原则升级（L1 → L2）

触发条件（同时满足）：
- 同一主题下积累了 ≥ 5 条相关经验
- 这些经验可以归纳为一条通用原则
- 该原则具有跨场景的复用价值

操作步骤：
1. 从 `memory/experiences.md` 中提取相关经验
2. 归纳为通用原则，按以下格式编写：

```markdown
### 原则 #[编号]
- **主题**：[原则主题]
- **原则内容**：[原则描述]
- **来源经验**：[引用的经验编号列表]
- **适用范围**：[通用/特定场景描述]
- **创建日期**：[日期]
```

3. 检查是否与 `memory/principles.md` 中已有原则矛盾
4. **向用户展示原则及来源经验，请求确认**
5. 用户确认后，追加写入 `memory/principles.md`
6. 更新 `memory/counters.md` 中相关经验的升级状态为"已升级为原则"

### 2.3 原则 → Skill 固化（L2 → L3）

触发条件（同时满足）：
- 原则可以被操作化为具体的步骤流程
- 该流程可以被独立执行，无需人工判断
- 该流程具有明确的触发条件

操作步骤：
1. 编写完整 Skill 文件，按以下格式：

```markdown
---
name: [Skill名称]
description: [一句话描述]
trigger: [触发条件描述]
created: [创建日期]
updated: [最后更新日期]
usage_count: 0
---

# [Skill名称]

## 触发条件
[详细描述何时应使用此Skill]

## 操作步骤
1. [步骤一]
2. [步骤二]
3. ...

## 注意事项
- [注意事项1]
- [注意事项2]

## 示例
[使用示例]
```

2. **向用户展示完整 Skill 文件，请求确认**
3. 用户确认后，保存到 `skills/[skill-name].md`
4. 在 `memory/counters.md` 中创建规则触发计数条目

### 2.4 Skill → 规则晋升（L3 → L4）

触发条件（同时满足）：
- 该 Skill 在计数器中被触发 ≥ 8 次
- 该 Skill 的操作步骤从未被违反或跳过
- 触发条件足够明确，无需人工判断

操作步骤：
1. 将 Skill 的核心行为提炼为简洁的行为规则
2. **判断规则范围**（详见 `copilot-meta-rules/global-local-rules.md`）：
   - 全局适用 → 全局规则
   - 仅当前项目 → 局部规则
3. **向用户展示规则内容及范围建议，请求确认**
4. 用户确认后：
   - **全局规则**：写入用户级全局规则文件（路径由用户配置）
   - **局部规则**：写入当前项目的 `.github/copilot-instructions.md` 或 `AGENTS.md`

> ⚠️ **重要：Skill → 规则晋升后，原 Skill 文件保留不删除。**
> Skill 可能被其他工作区复用。除非人类明确提出删除请求，否则 Agent 不得执行删除操作。

## 3. 人类主动沉淀

除了自动触发外，人类可以随时主动要求沉淀：

| 人类指令示例 | 触发流程 | 跳过的检查 |
|------------|---------|-----------|
| "把这条经验记下来" | 经验提取（L0 → L1） | 跳过计数阈值检查 |
| "这个可以升级为原则了" | 经验→原则升级（L1 → L2） | 跳过计数阈值检查 |
| "帮我写个Skill" | 原则→Skill固化（L2 → L3） | 跳过计数阈值检查 |
| "这条应该成为规则" | Skill→规则晋升（L3 → L4） | 跳过计数阈值检查 |

**人类主动沉淀同样遵循上述完整规则**：
- 必须按格式编写
- 必须展示给人类确认（即使是人类发起的，也需要最终确认）
- 必须更新计数器
- 必须执行多关卡审核（详见 `copilot-meta-rules/quality-assurance.md`）

唯一区别：人类主动沉淀**跳过计数阈值检查**，但其他所有步骤不可省略。

---

## 4. 沉淀操作的格式规范

所有沉淀产物必须包含以下元数据：

| 字段 | 说明 | 是否必须 |
|------|------|---------|
| 编号 | 唯一标识符（如 #0042） | 是 |
| 创建日期 | 首次创建的日期 | 是 |
| 来源 | 触发该沉淀的原始对话/经验 | 是 |
| 适用条件/范围 | 该知识适用的场景 | 是 |
| 升级状态 | 当前在沉淀路径中的位置 | 是 |

---

## 5. 沉淀操作的禁止事项

- ❌ 不得跳过人类确认步骤
- ❌ 不得省略格式化编写步骤
- ❌ 不得跳过计数器更新
- ❌ 不得跳过与已有知识的矛盾检查
- ❌ 不得自动删除任何沉淀产物（特别是 Skill 文件）
- ❌ 不得在未达到阈值时自动触发沉淀（人类主动沉淀除外）
