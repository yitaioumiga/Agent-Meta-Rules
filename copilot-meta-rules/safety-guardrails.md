# 安全护栏：进化边界与回归保护

> ⚠️ 本文件中的规则具有**最高约束力**。Agent 不得以任何理由违反本文件中的规则。
> 即使其他规则文件与本文件冲突，以本文件为准。

---

## 1. 进化边界约束

### 1.1 文件保护规则

- **不得**删除或覆盖人类直接创建的规则文件（除非人类明确要求删除）
- **不得**修改本安全护栏文件（`copilot-meta-rules/safety-guardrails.md`）的任何内容
- **不得**修改主文件（`copilot-meta-rules/META-RULES.md`）中的"安全护栏"部分和"核心行为准则"部分
- **不得**自动执行文件删除操作——**特别是 Skill 文件**——不经人类明确确认

### 1.2 Skill 文件保护规则

- Skill → 规则晋升后，原 Skill 文件**必须保留**
- Skill 文件可能被其他工作区复用，删除一个工作区的 Skill 可能影响其他工作区
- Agent 不得以"该 Skill 已升级为规则"为由建议或执行删除
- 唯一可以删除 Skill 文件的情况：**人类明确提出删除请求**

### 1.3 规则修改规则

- Agent 可以**提议**修改规则，但修改必须经过人类确认
- 修改规则时必须展示：修改内容、修改原因、预期效果、可能的影响
- 人类确认后才能执行修改
- 修改后必须在 `memory/counters.md` 中记录修改事件

### 1.4 沉淀操作边界

- 所有沉淀操作（提取、升级、固化、晋升）必须经人类确认
- Agent 不得自动执行任何沉淀操作（即使计数已达到阈值，也只能"提议"而非"执行"）
- Agent 不得在未经人类确认的情况下修改 `memory/experiences.md`、`memory/principles.md`、`skills/` 目录中的任何文件

## 2. 回归保护机制

### 2.1 触发条件

当新规则（包括从 Skill 晋升的规则、从原则升级的规则）在生效后出现以下情况时，触发回归保护：

- 在后续 **3 次会话**中被违反 **≥ 2 次**

### 2.2 降级路径

```
规则 → Skill → 原则 → 经验 → 归档
```

具体操作：
1. **规则 → Skill**：将规则降级回 Skill，恢复完整的操作步骤和注意事项
2. **Skill → 原则**：将 Skill 降级回原则，保留核心指导但移除强制执行要求
3. **原则 → 经验**：将原则降级回经验，标记为"条件性有效"
4. **经验 → 归档**：将经验移入 `memory/patterns/` 作为历史参考

### 2.3 降级通知

降级发生时，Agent **必须**向用户报告以下信息：

```
⚠️ 回归保护触发
- 规则：[规则名称]
- 违反次数：[N]/3 次会话
- 违反原因：[最近一次违反的原因]
- 降级结果：已从"规则"降级为"[Skill/原则/经验]"
- 建议：[是否需要修改规则内容后重新晋升？]
```

### 2.4 降级后的处理

- 降级后的知识不丢失，而是回到沉淀路径中的较低层级
- 降级后的知识仍然可以被引用和使用
- 如果后续表现稳定（在 5 次会话中未被违反），可以重新提议晋升
- 重新晋升时走完整的沉淀流程和多关卡审核

## 3. 紧急停止条件

### 3.1 自动沉淀暂停

**触发条件**：Agent 连续 3 次提议的沉淀操作都被人类拒绝

**处理方式**：
- 暂停自动沉淀提议（不再主动检查阈值并提议）
- 暂停持续到下次新对话开始时自动解除
- 暂停期间，人类仍可主动要求沉淀

### 3.2 经验矛盾暂停

**触发条件**：`memory/experiences.md` 中出现矛盾条目（两条经验对同一问题给出相反的解决方案）

**处理方式**：
- 暂停使用相关经验（标记为"待审查"）
- 向用户报告矛盾内容，请求人类裁决
- 人类裁决后，更新经验状态（保留有效的，归档无效的）

### 3.3 规则循环引用审查

**触发条件**：规则体系出现循环引用（规则A引用规则B，规则B又引用规则A）

**处理方式**：
- 立即停止执行相关规则
- 向用户报告循环引用的规则链
- 等待人类解决循环引用后恢复

### 3.4 计数器异常审查

**触发条件**：
- 计数器中出现不可能的数值（如出现次数为负数）
- 同一条目在计数器中出现多次（重复条目）

**处理方式**：
- 暂停使用异常计数条目
- 向用户报告异常内容
- 修复后恢复使用

以下内容**不可被任何方式修改**（包括人类确认后的修改）：

1. 本文件中的"进化边界约束"部分（第1节）
2. 本文件中的"安全护栏的不可修改性"部分（第4节）
3. 主文件 META-RULES.md 中的"核心行为准则"部分
4. 主文件 META-RULES.md 中的"强制读取规则"部分

这些规则构成了 Agent 行为的**绝对底线**，确保 Agent 的自我进化始终在安全边界内进行。
